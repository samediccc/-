# 需求规格：侧边导航栏组件

## 1. Purpose (目的)

设计并实现一个功能丰富的、可折叠的侧边导航栏组件。该组件是用户探索应用功能的核心入口。

## 2. Requirements (需求列表)

-   **整体结构**: 导航栏从上到下应划分为四个逻辑区域：
    1.  **快速导航 (Quick Navigation)**: 包含“地图”、“时间轴”、“地标”、“故事”四个按钮，每个按钮由一个图标和文字标签组成。
    2.  **时代切换 (Era Switcher)**: 包含“古代长安”和“现代西安”两个切换按钮。
    3.  **搜索 (Search)**: 包含一个文本输入框和一个搜索图标按钮。
    4.  **快捷功能 (Quick Actions)**: 包含“全屏”、“重置”等快捷操作按钮。
-   **样式**: 
    -   导航栏整体背景色应与页眉保持一致。
    -   按钮在悬停 (hover) 和激活 (active) 状态下应有明显的视觉反馈（如背景色变化或高亮）。
    -   每个区域之间应有分割线或视觉间距，以保持清晰的层次感。

## 3. Interaction / Logic (交互逻辑)

-   **导航跳转**: 点击“快速导航”中的按钮，应能将主内容区滚动或切换到对应的功能模块。
-   **时代切换**: 
    -   “古代长安”和“现代西安”是单选逻辑，任一时刻只有一个能被选中。
    -   被选中的按钮应保持高亮状态。
    -   点击切换时，需要触发一个全局事件（例如，通过 `CustomEvent`），通知其他组件（如地图模块）更新其显示内容以匹配所选时代。
-   **显隐控制**: 导航栏的显示和隐藏由外部的汉堡包按钮控制。在小屏幕上，导航栏应以抽屉或覆盖层（Overlay）的形式从左侧滑出。
