# 需求规格：SVG 交互式地图模块

## 1. Purpose (目的)

创建应用的核心可视化模块——一个基于 SVG 的交互式地图。该地图将用于承载地理信息、地标和路线，并允许用户进行基本交互。

## 2. Requirements (需求列表)

-   **技术选型**: 使用纯 SVG 技术进行绘制，不引入第三方地图库（如 Leaflet, Mapbox）。
-   **地图元素**: 地图需要包含以下几个图层 (SVG `<g>` 元素)，以便于管理：
    1.  `map-background`: 地图背景或轮廓。
    2.  `map-routes`: 道路或水系等线状元素。
    3.  `map-landmarks`: 地标点。
    4.  `map-labels`: 地名或地标标签。
-   **控件**: 在地图右下角提供三个独立的控制按钮：
    1.  放大 (`+`)
    2.  缩小 (`-`)
    3.  重置视图 (一个重置图标)
-   **样式**: 地图元素的颜色（如地标、路线）应能通过 CSS 变量进行定义，以便在切换“时代”时，可以通过 JavaScript 动态修改这些变量来改变地图的整体色调。

## 3. Interaction / Logic (交互逻辑)

-   **缩放 (Zoom)**: 点击放大/缩小按钮，或使用鼠标滚轮，可以平滑地缩放地图视图。
-   **平移 (Pan)**: 用户可以通过鼠标拖拽来平移地图。
-   **地标交互**:
    -   **悬停 (Hover)**: 当鼠标悬停在某个地标点上时，应显示该地标的名称作为提示（Tooltip）。
    -   **点击 (Click)**: 当用户点击某个地标点时，该地标应有高亮效果（如描边加粗或发光），并触发一个全局事件，将该地标的唯一 ID 传递出去，以供信息面板捕获并显示详情。
-   **重置**: 点击重置按钮，地图的缩放和平移状态应恢复到初始默认值。
