# 需求规格：应用整体页面布局

## 1. Purpose (目的)

本需求旨在定义“西安时空地图”应用的宏观界面布局，为后续组件开发提供一个清晰、统一的框架。AI 助手需根据此规范生成基础的 HTML 结构和 CSS 布局代码。

## 2. Requirements (需求列表)

-   **结构**: 采用经典的“页眉 (Header) - 主内容 (Main) - 页脚 (Footer)”三段式垂直布局。
-   **页眉 (Header)**:
    -   应始终固定在页面顶部。
    -   内部包含应用 Logo、标题“西安时空地图”和一个用于打开导航菜单的汉堡包按钮。
    -   背景色应采用深色调，与主题匹配。
-   **主内容 (Main)**:
    -   应占据除页眉和页脚外的所有剩余垂直空间。
    -   内部采用左右分栏布局：左侧为可折叠的侧边导航栏，右侧为核心内容区（用于放置地图、信息面板等）。
-   **页脚 (Footer)**:
    -   应始终固定在页面底部。
    -   包含版权信息 “© 2024 西安时空地图”。
-   **响应式**: 布局需要具备基本的响应式能力。在小屏幕设备（如手机）上，侧边导航栏应默认隐藏。

## 3. Interaction / Logic (交互逻辑)

-   页眉中的汉堡包按钮用于控制左侧导航栏的显示与隐藏。
-   主内容区的左右分栏比例应在桌面端保持稳定，例如左侧占 25%，右侧占 75%。
